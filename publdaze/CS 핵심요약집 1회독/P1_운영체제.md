# 🔑 운영체제 키워드 정리

## Ch1. 운영체제

### 1.1.1 운영체제란

- 운영체제
  - 소프트웨어 계층
  - 자원 관리
  - 사용자 사용 환경 제공
  - 윈도우, 맥OS, 리눅스, 유닉스

### 1.1.2 운영체제의 목적

- 처리 능력 향상
- 반환 시간 단축
- 사용 가능도 향상
- 신뢰도 향상

### 1.1.3 CPU와 메모리 구조

- CPU
  - 프로세서
- 메모리
  - 데이터 저장
  - 주 기억장치
    - 휘발성
    - 메인 메모리 (RAM)
  - 보조 기억장치
    - 비휘발성
    - SSD
    - HDD
  - 계층 구조
    - 레지스터
    - 캐시 메모리
    - RAM
    - 하드 디스크

### 1.1.4 커널과 시스템 콜

- 커널
  - OS 핵심 요소
  - 하드웨어와 프로세스 보안
  - 자원 관리
    - CPU 스케줄링
    - 메모리 관리
    - 입출력 관리
    - 파일 시스템 관리
  - 하드웨어 추상화
- 사용자 모드 ↔️ 시스템 콜 ↔️ 커널 모드

## Ch2. 프로세스

### 1.2.1 프로세스와 스레드

- 프로세스
  - 프로세스 메모리 영역
    - 스택
    - 힙
    - 데이터
    - 코드
  - 스택 오버플로
  - 힙 오버플로
- 스레드
  - 메모리 영역 공유

### 1.2.2 PCB

- PCB에 저장되는 정보
  - 포인터
  - 프로세스 상태
  - PID
  - PC
  - 레지스터
  - ...

### 1.2.3 프로세스의 생성

- 부모 프로세스 -> fork() -> 자식 프로세스
  - 반환 값
    - 부모 - 자식 PID
    - 자식 - 0

### 1.2.4 프로세스 상태도

- 생성
- 준비
  - 디스패치
- 대기
- 실행
  - 인터럽트
- 종료

### 1.2.5 멀티 프로세스와 멀티 스레드

- 동시성
  - 콘텍스트 스위칭
- 병렬성
- 멀티 프로세스
  - 오버헤드
  - IPC
- 멀티 스레드
  - 동기화

### 1.2.6 콘텍스트 스위칭

- 인터럽트 -> 콘텍스트 스위칭
- 콘텍스트 스위칭 과정
  - PCB에 저장
  - 레지스트에 로드
  - 유휴 상태
- 코드 이어서 실행
  - PC
  - 스택 포인터

### 1.2.7 프로세스 동기화

- 경쟁 상태
  - ex. 우유 문제
- 임계 영역
  - 동시 접근 방지 조건
    - 상호배제 기법
    - 진행
    - 항정된 대기
- 뮤텍스 (락킹 매커니즘)
  - 락
  - ex. 화장실 예
  - 바쁜 대기) 스핀락
- 세마포어 (시그널링 메커니즘)
  - ex. 화장실 예

### 1.2.8 교착 상태

- 교착 상태 발생 필요 충분 조건
  - 상호배제
  - 점유와 대기
  - 비선점
  - 환형 대기

### 1.2.9 스레드 안전

- 스레드 안전
  - 멀티 스레드 환경

### 1.2.10 IPC

- IPC 종류
  - 공유 메모리
  - 소켓
  - 세마포어
  - 파이프
  - 메시지 큐

### 1.2.11 좀비 프로세스와 고아 프로세스

- 좀비 프로세스
  - 자식 프로세스 종료 -> SIGCHLD -> wait() -> 회수 실패
- 고아 프로세스
  - 부모 PID 1(init 프로세스)로 변경

## Ch3. 스케줄링

### 1.3.1 스케줄링의 목적

- 공평성
- 효율성
- 안정성
- 반응 시간 보장
- 무한 연기 방지

### 1.3.2 스케줄링의 단계

- 장기 스케줄링
  - 잡 스케줄링 / 승인 스케줄링
- 중기 스케줄링
  - 스왑 아웃 -> 중단 상태
- 단기 스케줄링
  - CPU 스케줄링

### 1.3.3 스케줄링 알고리즘

- 평가 기준
  - CPU 사용률
  - 처리량
  - 응답 시간
  - 반환 시간
  - 대기 시간
- 비선점현
  - FCFS
  - SJF
    - 기아 상태
- 선점형
  - RR
  - SRTF
    - 기아 상태
  - 멀티 레벨

## Ch4. 메모리 관리 전략

### 1.4.1 논리 메모리와 물리 메모리

- 논리 메모리 영역(가상 메모리 영역)
- 물리 메모리 영역
- 논리 주소(가상 주소) -> 메모리 관리 장치(MMU) -> 물리 주소

### 1.4.2 연속 메모리 할당

- 고정 분할 방식
  - 단편화 문제 발생 가능
    - 외부 단편화
    - 내부 단편화
- 가변 분할 방식
  - 최초 적합
  - 최적 적합
  - 최악 적합

### 1.4.3 비연속 메모리 할당

- 페이징
  - 페이지 - 논리 메모리 영역
  - 프레임 - 물리 메모리 영역
  - 페이지 테이블
    - 프로세스의 페이지 정보
    - 페이지에 매핑하는 프레임 주소 값
  - 외부 단편화 해결
  - 내부 단편화 문제
- 세그먼테이션
  - 세그먼트로 분할
  - 세그먼테이션 테이블
    - 세그먼트별 시작 주소 (base)
    - 세그먼트 길이 (limit)
  - 외부 단편화 문제

## Ch5. 가상 메모리

### 1.5.1 가상 메모리란

- 메모리에 일부 로드
- 장점
  - 메모리 크기 제약 덜 받음
  - CPU 이용률과 처리율 높일 수 있음
  - 스와핑 횟수를 줄여서 프로그램 실행 속도 상승

### 1.5.2 요구 페이징

- 요구 페이징
  - 프로세스에 필요한 페이지만 메모리에 로드
- 패이지 폴트

### 1.5.3 스레싱

- 스레싱 - CPU 이용률 떨어지는 상황
- 워킹 세트
